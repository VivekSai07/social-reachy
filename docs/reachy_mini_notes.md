## Platform support

Linux, macOS, and **Windows** are all natively supported ‚Äî no Docker involved
anywhere in the official docs. GUI simulation (MuJoCo viewer) needs a real
display, which is another reason to run this natively rather than in a
container on Windows.

## Requirements

- Python 3.10‚Äì3.12 (3.12 recommended ‚Äî this project pins `.venv` to 3.12 via
  `uv`).
- `git` + `git lfs` if pulling apps/models from the Hub.

## Core concepts

- **Daemon**: background service that talks to the motors/sensors (or the
  MuJoCo simulator with `--sim`). Must be running before any SDK script or
  app can connect. Exposes a REST/WebSocket API at `http://localhost:8000`
  (docs at `/docs`).
- **`ReachyMini`**: the SDK client class. `with ReachyMini() as mini:` connects
  to whatever daemon is running (real robot or sim) ‚Äî application code is
  identical either way.
- **Motion**: `goto_target(...)` for smooth, ‚â•0.5s interpolated moves;
  `set_target(...)` for real-time/high-frequency control (tracking, games,
  10Hz+ loops). Controls `head` (6DOF Stewart platform pose), `antennas`
  (2 motors), `body_yaw`.
- **Safety limits** (SDK clamps automatically): head pitch/roll ¬±40¬∞, head
  yaw ¬±180¬∞, body yaw ¬±160¬∞, head-vs-body yaw delta ‚â§65¬∞.
- **Apps**: the daemon runs at most one app at a time, launched as a Python
  subprocess (`python -u -m <pkg>.main`). An app is a class extending
  `ReachyMiniApp` with a `run(reachy_mini, stop_event)` method. Structure,
  entry points (`[project.entry-points."reachy_mini_apps"]` in
  `pyproject.toml`), and HF metadata **must** be generated by the official
  CLI ‚Äî never hand-rolled.

## CLI: `reachy-mini-app-assistant`

```bash
# scaffold a new app (never create the folders by hand)
reachy-mini-app-assistant create <name> <path> [--template default|conversation] [--publish]

# validate structure/entry points/install-ability
reachy-mini-app-assistant check <path>

# publish an already-created app to Hugging Face Spaces
reachy-mini-app-assistant publish <path>
```

- `default` template: minimal `ReachyMiniApp` skeleton (what `apps/social_app`
  currently is).
- `conversation` template: forks the reference conversation app ‚Äî wires up
  VAD, LLM, TTS, and movement fusion. Swap to this if the prototype needs
  spoken dialogue (see `reachy_mini_conversation_demo` in Community
  Resources below).
- `--publish` creates a **public** Hugging Face Space immediately ‚Äî never run
  it without explicit user confirmation.
